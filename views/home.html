Public endpoints
GET /api/ping Liveness probe for load balancers and uptime monitors. GET /api/health Service health with Mongo, Redis, Postgres, Kafka, RabbitMQ, and Elasticsearch checks. GET /api/metrics Counts for budgets, expenses, tasks, customers, orders, and process stats. GET /docs OpenAPI explorer for REST endpoints. GET /api/graphql GraphiQL playground for the GraphQL schema.
Authenticated REST resources

Most CRUD endpoints require a Bearer token from the auth service.

GET /api/budgets Budgets collection.
GET /api/expenses Expenses across budgets.
GET /api/tasks Task queue state + history.
GET /api/orders Customer order records.
GET /api/customers Customer profiles.
GET /api/users User administration data.
GET /api/transactions PostgreSQL-backed transaction logs.
Tip: use POST /api/auth/login to obtain a JWT, then send it via Authorization: Bearer <token>.
Auth workflow

Use these endpoints to register users, sign in, and verify sessions.

POST /api/auth/register Create an account.
POST /api/auth/login Authenticate and receive a JWT.
POST /api/auth/verify-token Validate a token for API calls.
POST /api/auth/logout Invalidate a session token. 
curl -X POST /api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"user@example.com","password":"password"}'
Search and notifications

Expense search is powered by Elasticsearch. Notifications broadcast through the notification controller.

POST /api/search Search expenses (auth required).
POST /api/notifications/send Send a notification payload. 
curl -X POST /api/search \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{"query":"utilities"}'
GraphQL and gRPC

GraphQL is available for flexible queries, while gRPC powers high-throughput budget workflows.

GET /api/graphql GraphiQL interface with live schema introspection.
gRPC 0.0.0.0:50051 BudgetManager service (override with GRPC_PORT).
Data stores and pipelines
  • MongoDB: budgets, expenses, tasks, customers, orders, users.
  • PostgreSQL: transaction_logs ledger for transactions.
  • Redis: cached task status and API-level caching.
  • Elasticsearch: expense search index.
  • Kafka: task event streaming.
  • RabbitMQ: task queue ingestion.
Operational checks

Use these for monitoring and sanity checks in deployments.

GET /api/ping Confirms routing and runtime availability. GET /api/health Service dependency health for deployments. GET /api/metrics Counts and process telemetry for dashboards.
Environment hints

Configuration lives in .env. Update data source URLs before starting the server.

  • MONGO_DB_URI for primary document storage.
  • POSTGRES_URL for transaction logs.
  • REDIS_URL for cache and task status.
  • ELASTIC_SEARCH_URL for expense search.
  • RABBITMQ_URL and KAFKA_BROKER for async workflows.
